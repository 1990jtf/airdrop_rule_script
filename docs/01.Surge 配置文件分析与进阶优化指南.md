# Surge 配置文件分析与进阶优化指南

## 1. 核心问题修复：Discord 无法访问

**问题分析：** 配置文件中 `[Rule]` 部分存在本地直连规则与代理规则集的冲突。
在早期的配置中，`DOMAIN-SUFFIX,discord.gg,DIRECT` 被设置为了直连。由于 Surge 规则自上而下匹配，该规则优先命中，导致下方的 Discord 专属规则集被忽略，流量未能走代理。

## 2. 配置文件常规优化与清理

基于当前的配置文件结构，建议进行以下清理与完善，以提高效率和防错能力：

### 2.1 清理冗余规则
在 `[Rule]` 顶部存在相互冲突的无效规则：
```ini
IP-CIDR,45.78.63.164/32,DIRECT,no-resolve
IP-CIDR,45.78.63.164/32,Proxy,no-resolve # 此行为无效死代码
```
**建议：** 删除第二行 `Proxy` 规则，仅保留 `DIRECT` 以防止本地环回死锁。

### 2.2 补充 GeoIP 兜底规则
当前的 `FINAL` 规则直接导向代理，可能导致未被规则库覆盖的国内新应用或冷门域名走代理，拖慢网速并消耗流量。
**建议：** 在 `FINAL` 前增加基于 IP 归属地的直连判断。
```ini
# =====================
# 最终规则（兜底规则）
# =====================
GEOIP,CN,DIRECT
FINAL,Proxy,dns-failed
```

### 2.3 优化测速策略 (Proxy Group)
目前 `Auto` 策略组使用的是 `fallback`（故障转移），它只会按顺序检查，直到当前节点断线才会切换，无法保证延迟最优。
**建议：** 如果希望 Surge 自动挑选延迟最低的节点，改为 `url-test`：
```ini
Auto = url-test, "🇭🇰 香港", "🇸🇬 新加坡", "🇯🇵 日本", "🇺🇸 美国", url=[http://www.gstatic.com/generate_204](http://www.gstatic.com/generate_204), interval=600, tolerance=50
```

### 2.4 评估 IPv6 及 MITM 设置
* **IPv6：** 目前开启了 `ipv6 = true`。若日常遇到某些外网图片加载缓慢或断流，建议改为 `false`，因为国内部分宽带的 IPv6 国际路由优化较差。
* **MITM：** 开启了针对 `findermp.video.qq.com`（微信视频号）的抓包解密，但缺少配套的脚本 (`[Script]`)。若未使用相关去广告或下载脚本，建议删除该 `hostname` 以节省设备性能。

## 3. 性能极限压榨（进阶）

通过优化 DNS 和底层协议，可以大幅降低网页首屏加载的延迟（空白期）。

### 3.1 DNS 解析极速优化
将 `system` 加入 DNS 列表首位，让本地直连请求优先使用运营商/路由器 DNS，从而分配到距离最近的 CDN 节点；同时移除国外 DoH 以防止拖慢国内网页解析。
精简强制真实 IP 解析列表，减少代理域名的多余发包。

### 3.2 提升规则引擎匹配效率
Surge 的 `RULE-SET`（规则集）需要正则或逐条遍历，而 `DOMAIN-SET`（域名集）底层基于哈希表，匹配速度为 **O(1)**。
**建议：** 尽量将大型规则库替换为 `DOMAIN-SET`，并将高频应用（如 Telegram、Discord）规则置顶。

### 3.3 连接与容灾超时控制
在弱网环境下，缩短 TCP 握手超时时间能让 Surge 更快放弃失效节点并触发切换，避免长时间卡顿。
