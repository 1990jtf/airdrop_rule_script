在 Surge 的语境里，`dns-server` 参数配置的，其实就是专门为你的本地网络环境（直连流量）提供解析服务的“上游 DNS 服务器”。

为了让你彻底弄懂 Surge 处理 DNS 的机制，我们可以把网络请求分成“两派”来看：

# 1. 直连流量（命中 DIRECT 规则的）
当你访问淘宝、微信、B 站这些国内服务时，Surge 需要知道它们的真实 IP 地址才能去连接。

这时候，Surge 就会去问你在 dns-server 里填写的那些服务器（比如阿里 DNS 223.5.5.5、腾讯 DNS 119.29.29.29 或者你的宽带路由器 system）。

由于这些都是国内最好的公共 DNS，它们能以最快的速度返回离你最近的国内服务器 IP（也就是 CDN 节点），让你实现秒开。

# 2. 代理流量（命中 Proxy 规则的）
当你访问 Google、YouTube、Discord 等被墙的网站时，Surge 根本不会去用你 dns-server 里配置的那些 DNS。
## 远端解析（Remote DNS）
Surge 非常聪明，它会把你要访问的“域名”（比如 discord.com）打包，直接顺着加密隧道扔给你在海外的代理节点（比如你的日本 BWG 节点）。
由日本的节点在当地去请求解析 discord.com，获取无污染的 IP 并建立连接。这个过程被称为远端解析（Remote DNS）。

💡 这就是为什么我之前强烈建议你把 8.8.8.8 和 1.1.1.1 从 dns-server 里删掉的原因：

如果你的 dns-server 里混进了 8.8.8.8：
对于FQ： 没用，因为FQ根本不查本地 DNS。
对于国内直连： 如果 Surge 刚好抽风用 8.8.8.8 去查淘宝的 IP，8.8.8.8 可能会因为地理位置判定，给你返回一个位于美国或者新加坡的淘宝 CDN 节点，导致你本来可以直连秒开的网页，加载变得极其缓慢。

总结来说，dns-server 就是纯纯的“国内直连导航员”，给它配最纯正的国内大厂 DNS 和 system 就对了！
